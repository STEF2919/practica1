LOS FRAMEWORKS MAS USADOS 2023

LARAVEL:

Este es un frameworks popular de PHP, que permite el desarrollo de aplicaciones web totalmente personalizadas de alta calidad. Laravel tiene un enfoque bastante moderno y ofrece muchas utilidades potentes a los desarrolladores, que permiten agilizar el desarrollo de las aplicaciones web y su mantenimiento.

Caracteristicas

  • Su motor de plantilla, llamado Blade.
  • Su arquitectura, conocida como (Modelo-Vista-Controlador).
  • Gratuito y código abierto.
  • Base de datos y migraciones.
  • Librerías y modularidad

Ventajas

  • Muy fácil de usar y aprender.
  • Su documentación.
  • Facilita el mantenimiento-actualización de aplicaciones.
  • Es gratuito.
  • Acelera el desarrollo Web.

Desventajas

  • Algunas librerías dependen del framework Sysmfony.
  • Es muy lento.
  • Sólo funciona con versiones de PHP superiores a la 5.4.
  • La curva de aprendizaje es lenta
  • Acelera el desarrollo Web.

DJANGO:

Es un marco web Python de alto nivel que fomenta un desarrollo rápido y un diseño limpio y pragmático, también facilita la creación de mejores aplicaciones web de forma más rápida y con menos código y eficiente.

Caracteristicas

  • Seguro
  • Escalable.
  • Portatil.
  • Versatil.
  • Rentabilidad

Ventajas

  • Proporciona una estructura sólida para el desarrollo de aplicaciones web.
  • Reduce la cantidad de código necesario para escribir una aplicación web.
  • Facilita trabajo en equipo, estándizar el código-herramientas utilizadas.
  • Su ORM (Mapeo de objeto racional) es sencillo y maravilloso.
  • Compatible como muchos frameworks.

Desventajas

  • Es monólito (antiguo).
  • Es lento y maneja peticiones de una en una.
  • Su Mapeo de objeto racional no es asíncrono ni tampoco el más veloz.
  • Curva de aprendizaje moderada.
  • Menos flexible.

ANGULAR:

Está basado en el lenguaje JavaScript- desarrollado en TypeScript por los programadores de Google, su nombre «Angular» proviene del paréntesis angulares(>>) que se utilizan en HTML, la misma que sirve para el desarrollar aplicaciones web de estilo Single Page Aplicación (SPA), Progressive Web App (PWA) tanto para versiones móviles como de escritorio.

Caracteristicas

  • El uso de Document Object Model(DOM) para representar el proyecto desde nodos.
  • Enlace de datos.
  • Compatabilidad móvil y de escritorio.
  • Productividad.
  • Velocidad y rendimiento.

Ventajas

  • Alta Calidad de la aplicacion.
  • Desarrollo multiplataforma.
  • Proceso de desarrollo web más rápido.
  • Aplicaciones web mas ligeras.
  • Código legible y comprobable.

Desventajas

  • Es dificil de entender.
  • No están flexible como otros frameworks.
  • Dependiente del servidor.
  • rendimiento.
  • Curva de aprendixaje pronunciada.

REACT:

Una biblioteca JavaScript para crear interfaces de usuario. ReactJS se utiliza ampliamente para crear aplicaciones web dinámicas e interactivas,Desarrollado por Facebook, ReactJS es una biblioteca JavaScript de código abierto muy extendida.

Caracteristicas

  • Composición de componentes..
  • Desarrollo Declarativo Vs Imperativo.
  • Ideal para aplicaciones de alta demanda.
  • Permite el desarrollo de aplicaciones móviles..
  • Ciclo de vida de los componentes.

Ventajas

  • Fácil de aprender. React es mucho más fácil de aprender debido a su simplicidad en términos de sintaxis.
  • Alto nivel de flexibilidad y máxima .
  • DOM virtual (Document Object Model) que convierte los documentos en formato HTML, XHTML o XML en una estructura de árbol más manejable para los navegadores mientras analiza los diferentes elementos de la aplicación.
  • Marco muy ligero, ya que los datos del lado del usuario pueden presentarse simultáneamente en el servidor.
  • La migración entre versiones suele ser muy sencilla, ya que Facebook proporciona «codemods» (cambios de código) para automatizar gran parte del proceso..

Desventajas

  • Falta de documentación oficial. La alta velocidad de desarrollo de React JS deja poco espacio para una documentación adecuada.
  • No hay un patrón de desarrollo.
  • Se necesita mucho tiempo para dominarlo, así como un profundo conocimiento de cómo integrar las interfaces de usuario en los frameworks MVC.
  • Juego de herramientas incompleto
  • Enfoque no vigilante en la interfaz del usuario

VUE.JS

Un framework JavaScript progresivo para construir interfaces de usuario.Tambien es conocido por su sencillez y facilidad de uso. Se centra en la creación de componentes reutilizables y modulares.

Caracteristicas

  • Orientado a componentes.
  • Es muy flexible y adaptable se puede integrar de a poco en proyectos de diferentes tamaños y complejidades.
  • Tiene una gran documentación y comunidad.
  • Trabaja con JavaScript.
  • Usado para aplicaciones de escritorio y móviles cuando se le combina con el framework Electron.

Ventajas

  • Su pequeño tamaño: puede que esto no sea una gran característica,pero los 18 KB que pesa este framework lo hacen ideal para una descarga rápida y poder almacenarlo en equipos de baja memoria, impactando de manera positiva tu SEO y UX.
  • Desglosar los componentes en archivos individuales,cuando se crea una aplicación o página web con VueJS, cada pieza de esta se divide en componentes individuales, representados como elementos encapsulados en su interfaz.
  • Se pueden escribir en HTML, CCS y JavaScript.
  • Fácil de aprender y utilizar, más amigables a la hora de ser utilizado por desarrolladores que van comenzando o personas que tienen entusiasmo por aprender lo básico de este campo.

Desventajas

  • La barrera del lenguaje,a la hora de acceder a ella desde occidente, el hecho de no saber hablar este idioma puede ser un impedimento para conocer sobre actualizaciones y mejoras.
  • No cuenta con apoyo para grandes proyectos debido a que es un Framework joven.
  • Ausencia de documentación oficial.
  • Requiere mucho demasiado tiempo para dominarlo.
  • No existe un estandár de desarrollo

SPRING

Un framework basado en Java para crear aplicaciones empresariales.Es una plataforma Java de código abierto.Tambien es conocido por su modularidad, su compatibilidad con la seguridad empresarial y su potente ecosistema de plugins y extensiones.Un framework, es un conjunto de herramientas y librerías que facilitan el desarrollo de una aplicación, permitiendo la reutilización de código previamente preparado para la ejecución de cierta tarea.Es decir se trata de un marco de trabajo compuesto de herramientas y utilidades ideales para la creación de aplicaciones web mediante el lenguaje programación Java.

Caracteristicas

  • Permite la creación de rutas dinámicas como para vistas,como para los entornos que lo requieran.
  • Trabaja perfectamente con el paradigma MVC (Modelo Vista Controlador).
  • Soporte para el Desarrollo de aplicaciones Reactivas con Spring WebFlux.
  • Para la creación y consumo de APIs desde dispositivos móviles
  • Sustenta el inicio de sesión que permite a las aplicaciones el acceso limitado a cuentas de usuario a través de tokens dinámicos.

Ventajas

  • Es muy versátil,pues se divide en módulos diseñados para solventar diferentes necesidades.
  • Desarrollo web sobre REST API spring hace gestión de la petición,realiza el análisis correspondiente de datos y luego decide en qué clase de Java procesará los datos.
  • Permite el desarrollo de aplicaciones flexibles y escalables.
  • Promueve una alta cohesión

Desventajas

  • A diferencia de frameworks como Django, Spring no es tan rápido y eficiente.
  • La falta de soporte nativo a las imágenes con GraalVM si la aplicación usa la API Java Reflection, que nos permite descubrir código en tiempo de ejecución
  • Depende de bibliotecas externas lo que puede aumentar la complejidad de uso y sobrecarga de la aplicación
  • Es muy dificil de aprender.
  • Por cada servicio que tenemos debemos configuarlo en un xml.
